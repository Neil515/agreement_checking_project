# 📘 README：AI 合約風險條款摘要工具助手

這是一個幫助大家快速看懂「合約」或「網站服務條款」裡面藏有什麼風險的小工具。

很多人每天都會點「我同意」，但實際上根本沒時間看完那些密密麻麻的條款。這個工具的目的是： 👉 幫你找出裡面可能會讓你吃虧的內容，並用簡單的話解釋給你聽。

---

## 🧩 這個工具是做什麼用的？

* 讀取條款（例如網站的使用者協議、隱私政策）
* 幫你切成一條一條的句子
* 針對每一條條款判斷「是否有風險」，並標出「風險等級」：高、中、低
* 最後產出一份簡單易懂的摘要報告

---

## ⚙️ 工具分成哪些階段開發？

### ✅ 第一階段：先做 Chrome 外掛

* 使用者在看條款網頁時，只要點一下外掛按鈕
* AI 就會幫你分析整篇條款，標出潛在風險

### 🧠 第二階段：加入 AI 判斷邏輯 + 使用者回饋功能

* 讓 AI 根據「關鍵字」與「句子特徵」來判斷風險
* 使用者可回饋 AI 的判斷對不對，幫助 AI 變得更聰明

### 🌐 第三階段：做成網站＋開放 API

* 讓企業或律師可以整合進自己系統
* 支援 PDF、Word 或純文字的合約分析

---

## 📊 我們的風險分類方式是什麼？

目前分為 8 種常見類型（可擴充）：

1. 單方面修改條款
2. 不可取消／不易解約
3. 永久授權
4. 收費模糊或未載明
5. 過度限制使用權利
6. 使用者需承擔高額賠償責任
7. 資料可能被分享或販售
8. 條款只受特定國家法律保護

每一類型會對應一組常見句型（關鍵語句），讓 AI 可以學會判斷。

## 🔍 資料從哪裡來？

我們會收集：

* 公開網站條款內容（例如 Google、Facebook、Zoom）
* 合法開放的資料集（ToS;DR、GitHub 合約集、Kaggle）
* 使用者手動貼上條款內容
* 未來可開放使用者上傳 PDF 或 Word 文件

---

## 🔧 我們會怎麼訓練 AI？

1. 先建立一批真實條款（50～100 條）並手動標註風險分類
2. 用這些當作 AI 的訓練範例，設計適合的提問方式（Prompt）
3. 每次讓 AI 分析一條新條款時，就依照這些標準回答
4. 使用者可回饋 AI 分析結果（對或錯），我們持續優化模型
